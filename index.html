Process

From Idea to Execution

Planning

I plan every project and this is arguably the most iportant part. This involves understanding the general and specific goals of the application. This also involves creating a timeline for internal and external deliverables. Creating this guide with tangible goals gets me excited and energized, making for an enjoyable development experience.

Coding

Using the plan, I translate the requirements into clean and clear code. Currently, I focus on Ruby on Rails, but I am open to any technology and ready to learn new languages, libraries, or frameworks. My goal is to code an efficient, adaptable, scalable application ready for real-world-use.

Improving

Throughout the development process and after launch, I engage in ongoing testing. I review the code and engage non-technical users to test-drive the application for usability. Identifying problems early during development means they can be fixed with less time and effort. I aim to create applications that serve a purpose, delivering results when expected, as expected.
